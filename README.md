# Overtime 초과근무 기록 시스템

Google Sheets를 데이터 저장소로 사용하는 초과근무 기록용 Vercel 웹앱입니다.

## 주요 기능

- 사용자 계정 로그인
- 회원가입, 아이디 찾기, 비밀번호 재설정
- 사용자별 월 시트 분리
- 월 마감/잠금 및 비밀번호 재입력 후 수정 허용
- 날짜별 메모 저장: 월 시트의 `H`열 사용
- 사유 카테고리 버튼: 회의, 생활지도, 방과후, 행사, 상담, 기타
- 지금 시작 / 지금 종료 빠른 저장
- 달력 날짜 선택 시 상세 기록과 해당 날짜 수정 이력 표시
- 방과후 요일 설정 저장
- 월별 CSV 다운로드 및 PDF 인쇄
- 수정 이력 기록: `수정이력` 시트 자동 생성
- PWA 설치 지원

## 첫 계정 만들기

앱 배포 후 첫 회원가입 계정은 자동으로 `admin` 역할이 됩니다.
요청한 계정은 앱 첫 화면의 회원가입에서 다음 값으로 직접 생성하면 됩니다.

- 아이디: `ahl0871`
- 비밀번호: 사용자가 지정한 비밀번호
- 복구코드: 아이디 찾기/비밀번호 재설정에 사용할 코드

비밀번호와 복구코드는 Google Sheet에 원문으로 저장되지 않고 해시로 저장됩니다.

## 추가 회원가입

추가 사용자는 회원가입 화면에서 아이디, 이름, 비밀번호, 복구코드를 입력해 바로 만들 수 있습니다.

## 사용자별 시트 규칙

- 첫 관리자 계정은 기존 월 시트(`26-5` 같은 이름)를 그대로 사용합니다.
- 일반 사용자의 월 시트는 내부적으로 `아이디__26-5` 형식으로 만들어집니다.
- 화면에는 모든 사용자에게 `26-5`처럼 월 이름만 보입니다.

## Vercel 환경변수

필수:

- `GOOGLE_CLIENT_EMAIL`: Google 서비스 계정 이메일
- `GOOGLE_PRIVATE_KEY`: Google 서비스 계정 private key
- `GOOGLE_SHEET_ID`: 연결할 Google Spreadsheet ID

권장:

- `APP_SESSION_SECRET`: 로그인 토큰 서명용 비밀값

선택:

- `TEMPLATE_SHEET_NAME`: 기본값 `양식`
- `DB_SHEET_NAME`: 기본값 `통합DB`
- `HISTORY_SHEET_NAME`: 기본값 `수정이력`
- `USERS_SHEET_NAME`: 기본값 `사용자`
- `LOCKS_SHEET_NAME`: 기본값 `월마감`

## Google Sheet 구조

월별 시트는 다음 열을 사용합니다.

| 열 | 용도 |
| --- | --- |
| A | 날짜 |
| C | 시작 시간 |
| D | 종료 시간 |
| E | 아침 초과 |
| F | 오후 초과 |
| G | 일계 |
| H | 메모 |
| J | 요약 값 |
| K | 방과후 요일 |
| L | 방과후 선택 여부 |

앱이 자동으로 관리하는 시트:

- `사용자`: 계정, 해시된 비밀번호, 해시된 복구코드
- `월마감`: 사용자별 월 마감 상태
- `수정이력`: 저장, 마감, 시트 생성 등의 작업 이력

`양식` 시트가 새 월 시트의 템플릿입니다. 새 월 이름은 `26-5` 또는 `2026-5` 형식으로 입력합니다.
